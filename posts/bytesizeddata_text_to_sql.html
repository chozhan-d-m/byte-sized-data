When it actually works and when to walk away

Scenario Action Why
Narrow domain, 5–10 curated views, well-named columns Use it LLM has enough context; failure surface is small
Self-service analytics for business users on clean warehouse Use it High value, low risk if semantic layer is in place
100+ table raw schema, abbreviated names, no documentation Avoid Model will hallucinate and fail unpredictably
Financial or executive reporting with no human review step Avoid Silent wrong answers are unacceptable here
Developer / analyst copilot with SQL review before execution Use it Human in the loop catches the hard failure cases
Complex analytics: window functions, multi-level GROUP BY With caution Performance drops sharply; needs validator layer
